# ComfyUI Creative Lab — 国际化 (i18n) 指南

本文说明 CLab 如何与 ComfyUI 自带的 `locales` 机制配合，以及**自定义侧边栏 / 工具栏**文案的维护方式。

---

## 1. 两套文案，不要混用

| 范围 | 数据从哪来 | 谁消费 |
|------|------------|--------|
| **Comfy 已接入能力** | `locales/<语言>/{main,nodeDefs,settings,commands}.json` 经后端合并 | Comfy 前端：节点定义、扩展设置项、命令、`settingsCategories` 等 |
| **CLab 自定义 DOM**（面板按钮、模块 hint、下拉占位、Toast 等） | 同上 **`main.json` 内的 `clabUi` 对象** | `web/clab_i18n.js` 通过 **`GET /i18n`** 拉取后，由业务代码调用 `clabT()` / `clabTf()` |

要点：

- `locales` **不会**自动替换你在 `innerHTML`、模板字符串里写死的文字；必须在 JS 里**显式**调用 `clabT('xxx.yyy')`。
- **单一维护源**：CLab 面板相关翻译只增改各语言目录下 **`main.json` 的 `clabUi`**（与 Comfy [自定义节点 i18n 文档](https://docs.comfy.org/custom-nodes/i18n) 所列语言对齐），不要在多个文件重复写整句文案。
- **开发与文案基准（简体中文）**：新增/修改界面文案时，**先在 `locales/zh/main.json`（及同语种的 `settings.json`、`nodeDefs.json` 等）落笔**，再派生到 `en`、`zh-TW` 与其它语言。`en` 是国际用户常用界面语言，但是作为**镜像**而非维护用的「第一母语」；`clabT` 在某语言缺键时会**先回退 `zh`，再回退 `en`**，与这一流程一致。
- **首屏与切换**：`ui_panel` 建面板时已用 `clabT` 写入顶栏；`attachDataIOEvents` 注入的导入/导出下拉在语言切换时由 **`refreshInjectedDataIoLabels(panel)`**（`action_data_io.js`）统一刷文案，并与 `applyClabPanelStaticI18n` 一起在 `initClabLocaleWatch` / `performRender` 末尾调用。
- **主菜单按钮**：顶栏 **CLab** 按钮文案为 `menu.brand`，`title` 为 `menu.openTitle`；随语言变化由 `refreshClabMenuButtonTitle()` 更新。
- **节点拾取菜单**：未分组节点在内部使用常量 `CLAB_NODE_MENU_UNGROUPED`（`ui_utils.js` 导出），界面标题仍用 `tree.ungroupedLabel`；Comfy 节点无 `category` 时分组标题用 `tree.uncategorizedLabel`。

官方自定义节点 i18n 说明（节点 / 设置 id 与 key 规则）可参考：[Comfy 文档 · Custom Nodes i18n](https://docs.comfy.org/)（站内「Custom Nodes i18n」章节）。

## 2. 目录结构（插件根目录）

本仓库 **已与 Comfy 官方支持的语言代码对齐**，在 `locales/` 下为每种语言维护一套 JSON（**键路径与嵌套结构以 `zh/main.json` 为权威**，各语言目录须具备相同键树，仅字符串值不同）：

| 目录名 | 语言 |
|--------|------|
| `en` | English |
| `zh` | 简体中文 |
| `zh-TW` | 繁体中文 |
| `ja` | 日本語 |
| `ko` | 한국어 |
| `ru` | Русский |
| `fr` | Français |
| `es` | Español |
| `ar` | العربية |

```
ComfyUI-CreativeLab/
├── locales/
│   ├── en/
│   │   ├── main.json      # settingsCategories + clabUi …
│   │   ├── nodeDefs.json
│   │   ├── settings.json
│   │   └── commands.json
│   ├── zh/   … (同上四文件)
│   ├── zh-TW/ …
│   ├── ja/    …
│   ├── ko/    …
│   ├── ru/    …
│   ├── fr/    …
│   ├── es/    …
│   └── ar/    …
└── web/
    ├── clab_i18n.js       # 请求 /i18n、解析 clabUi、工具函数
    └── main.js            # setup 早期 await loadClabI18nBundle()
```

后端（Comfy `CustomNodeManager`）会扫描各自定义节点的 `locales`，把 `main.json` 的内容并到对应语言节点下，并挂载 **`/i18n`** 接口返回整包 JSON。

## 3. `main.json` 里 CLab 用到的结构

### 3.1 `settingsCategories`

与 `web/clab_settings.js` 里 `registerExtension({ settings })` 的 **`category`** 数组分段一致，供 Comfy **设置界面**翻译分组标题。子级键需与 `category` 里出现的字符串一致（含第三段如 `Glow`、`Shortcut` 等）。

### 3.2 `clabUi` — 面板与工具栏文案（核心）

所有给 **`clabT('a.b.c')`** 用的路径都在根对象的 **`clabUi`** 下，与 `settingsCategories` 平级。访问规则：**点号路径 = JSON 嵌套**，例如 `clabT('module.boolTrue')` ↔ `clabUi.module.boolTrue`。

#### `clabUi` 命名空间与代码对照表

| 命名空间 | 典型用途 | 主要消费代码 |
|---------|----------|----------------|
| `toolbar` | 顶栏：新建任务/模块、运行、运行全部、批量次数、锚点按钮、卡片宽度控件 | `ui_panel.js`（首屏模板）、`clab_i18n.js` → `applyClabPanelStaticI18n` |
| `config` | 配置节点拆分按钮旁「数据维护」下拉（维护标题、清理、重同步） | `comp_toolbar.js`（首次注入）、`applyClabPanelStaticI18n` |
| `menu` | 主菜单 **CLab** 按钮：`brand`（可见文字）、`openTitle`（含 `{key}` 快捷键） | `main.js`、`clab_i18n.js` → `refreshClabMenuButtonTitle` |
| `module` | 输入模块：占位、上传、布尔 **On/Off（boolTrue/boolFalse）**、hint 等 | `module_input.js` |
| `moduleToolbar` | 悬浮动态工具栏：输入/输出、绑定、比例、格式刷等 | `action_module_config.js` |
| `fillMode` / `ratio` | 填充模式 / 自定义比例 **展示名**（存储值仍可为中文，见 §6） | `clabFillModeLabel`、`clabRatioLabel` |
| `tree` | 节点菜单：`ungroupedLabel`、`unnamedGroup`、`uncategorizedLabel` | `ui_utils.js`（`getCustomNodeMenuHTML`、`getMultiNodeMenuHTML`）、`clabTreeTitle` |
| `select` | 清除关联/绑定、无参数提示 | `ui_utils.js` |
| `context` | 右键菜单各项与扫描/同步 Toast | `comp_contextmenu.js` |
| `state` | 配置节点已存在 / 已创建 | `state_manager.js` |
| `card` | 删除任务卡片按钮 `title` | `comp_taskcard.js` |
| `dataIo` | JSON 导入导出下拉、ZIP、整理、`alert` / Toast | `action_data_io.js`、`refreshInjectedDataIoLabels` |
| `run` | 「运行」「运行全部」相关 `alert` | `action_run_executor.js` |
| `batch` | 跨任务同步参数、删除相同模块的 `confirm`（`{match}`、`{type}`） | `action_batch_sync.js` |
| `binding` | 拾取节点模式下的 `showBindingToast` | `action_binding.js` |
| `toasts` | 维护中心进度、**节点校验/后端报错** 等全局 Toast | `comp_toolbar.js`、`event_execution.js`、`api_injector.js` |
| `alerts` | 需先选卡片等阻塞式 `alert` | `comp_toolbar.js` 等 |
| `settingsUi` | 设置里「恢复全部默认」的 `confirm` | `clab_settings.js` |
| `media` | 音视频控件 `title`、画中画/全屏、未知文件名、下载链接文案 | `media_audio.js`、`media_video.js`、`media_file.js` |

结构示例（节选）：

```json
{
  "settingsCategories": { ... },
  "clabUi": {
    "toolbar": { "addCard": "…", "run": "…", ... },
    "menu": { "brand": "CLab", "openTitle": "…" },
    "module": { "boolTrue": "…", "boolFalse": "…", ... },
    "tree": { "ungroupedLabel": "…", "uncategorizedLabel": "…", ... },
    "dataIo": { ... }
  }
}
```

新增界面文案时：

1. 在 **`locales/zh/main.json` → `clabUi`**（以及对应的 **`settingsCategories`**、必要时 `settings.json` / `nodeDefs.json`）增加**中文**文案与键名。
2. 在 **`locales/en/main.json`** 及其它语言目录中补上**同名键**的译文（`en` 视为完整镜像，勿独占新键）。
3. 在组件里使用 `clabT('新命名空间.键')`，避免硬编码自然语言字符串。

## 4. 前端 API（`web/clab_i18n.js`）

| 函数 | 作用 |
|------|------|
| `loadClabI18nBundle()` | `fetch('/i18n')`，缓存全语言包；须在扩展 `setup` 中 **`await`**（见 `main.js`） |
| `getClabLocale()` | 根据 Comfy 语言设置等归一化为 **`en` / `zh` / `zh-TW` / `ja` / `ko` / `ru` / `fr` / `es` / `ar`**（与上表目录名一致；未识别时回退 `en`） |
| `clabT('group.key')` | 取当前语言下 `clabUi` 的字符串；缺键时回退 **`zh`**，再 **`en`**，再扫描其余语言包，最后返回 key |
| `clabTf('group.key', { name, count })` | 将文案中的 `{name}`、`{count}` 等占位符替换为传入值 |
| `clabFillModeLabel(stored)` | 填充模式：**存储值**仍为中文（`显示全部` / `填充` / `拉伸`），**仅标签**随语言切换 |
| `clabRatioLabel(stored)` | 比例：`'自定义比例'` 映射翻译，其余如 `16:9` 保持原样 |
| `clabTreeTitle(name)` | 分组树里将系统占位名 **`未命名组`** 译为当前语言；用户自定义组名保持原文 |
| `applyClabPanelStaticI18n(panel)` | 按 id 刷新 CLab **面板内**顶栏按钮与配置维护下拉的文案（不处理主 Comfy 菜单） |
| `initClabLocaleWatch(cb)` | 定时比对语言；变化时执行回调（见 §5） |
| `refreshClabMenuButtonTitle()` | 更新 **Comfy 主菜单** 中带 CLab 图标按钮的 **`title`**（`menu.openTitle`）与 **可见 `<span>`**（`menu.brand`） |
| `clearClabI18nCache()` | 调试：清空内存缓存后再请求 `/i18n` |

### 4.1 其它模块中的 i18n 辅助 API

| 符号 | 位置 | 作用 |
|------|------|------|
| `refreshInjectedDataIoLabels(panel)` | `web/components/actions/action_data_io.js` | 仅更新已存在的 **JSON 导入/导出** 下拉中的文案（分组标题、条目、`title`），**不重建 DOM、不重绑事件** |
| `CLAB_NODE_MENU_UNGROUPED` | `web/components/ui_utils.js`（export） | LiteGraph 节点**未落入任何分组框**时的内部树键；界面展示仍用 `clabT('tree.ungroupedLabel')` |

语言检测会尝试 Comfy 设置中的 `Comfy.Locale` 等；开发时可用 `window.__CLAB_FORCE_LOCALE__` 设为上述任一类语言代码（如 `'ja'`、`'zh-TW'`）做强制验证。繁体界面下 `pickLangEntry` 会优先使用 `zh-TW` 再回退 `zh`；**单键缺译**时 `clabT` 依次尝试 **当前语言 →（繁体链）→ 简体中文 `zh` → 英文 `en` → 其它语言包**。

## 5. 文案刷新时机（避免换语言后漏更新）

以下顺序保证 **顶栏 + 注入式数据菜单 + 主菜单按钮** 与当前语言一致：

1. **扩展启动**（`main.js`）：`await loadClabI18nBundle()` → `setupUI()`。  
2. **创建面板**（`ui_panel.js` → `createPanelDOM`）：顶栏 HTML **直接嵌入** `clabT(...)`，首屏即为当前语言。  
3. **`setupUI` 收尾**：`applyClabPanelStaticI18n(panel)` → **`refreshInjectedDataIoLabels(panel)`**（导入/导出下拉已由 `setupStaticToolbarEvents` → `attachDataIOEvents` 插入）。  
4. **每次 `performRender`**：`applyClabPanelStaticI18n` 之后再次调用 **`refreshInjectedDataIoLabels`**（防止某次重绘替换顶栏区域后文案陈旧）。  
5. **语言切换**（`initClabLocaleWatch` 回调）：`applyClabPanelStaticI18n` → **`refreshInjectedDataIoLabels`** → **`refreshClabMenuButtonTitle()`** → `document.dispatchEvent('clab_render_ui')` 触发整表重绘（动态工具栏、卡片内模块等重新 `generate*HTML`，自然带上新 `clabT`）。

**注意**：已打开的 **音视频播放器** 等若未触发整卡重绘，其局部 `title` 可能仍为旧语言；完全重开面板或触发一次会重渲卡片的操作即可。

---

## 6. 修改文案后的注意点

1. **重启 ComfyUI**：服务端对翻译表有 **`@lru_cache`**，只改 JSON 不重启可能不会更新 `/i18n`。
2. **浏览器强刷**：避免旧 `/i18n` 被缓存（视部署而定）。
3. **占位符与 HTML**：若在 `clabUi` 里含 `"` 等字符，嵌入 `placeholder="..."` 时需转义或避免引号冲突。

---

## 7. 与旧工作流数据兼容

部分内部状态仍使用**固定中文字符串**作为存储值（例如 `fillMode`、`ratio === '自定义比例'`），以便旧工作流 JSON 不降級。界面上通过 `clabFillModeLabel` / `clabRatioLabel` 只改**展示**，不改 `data-value` 与序列化逻辑。若未来要做「全英文存储」，需另做迁移方案，不在当前指南范围。
