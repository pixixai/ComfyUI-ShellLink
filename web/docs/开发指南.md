# ComfyUI Creative Lab (CLab) 开发者指南

本文档面向对 CLab (Creative Lab) 插件进行二次开发或维护的工程师，详述了经过**模块化极限重构**与**全局命名空间规范化**后的文件职责、系统架构以及核心函数的调用说明。

## 1. 架构概述

CLab 采用了完全解耦的组件化前端架构（MVC / 乐高模式）。数据层 (Model)、视图层 (View) 和控制交互层 (Controller / Actions / Events) 被严格分离，并引入了极高标准的渲染保护机制：

- **数据流向**：视图组件修改 `ui_state.js` 中的中心化数据 -> 触发无感保存 -> 执行 DOM 重绘或微创替换 -> 将 JSON 序列化并存入 ComfyUI 画布上的 `CLab_SystemConfig` 节点。
- **媒体保险库 (Media Vault) 防闪烁机制**：为了对抗 ComfyUI 高频的数据同步引起的 DOM 重建，系统在 `comp_modulearea.js` 和 `ui_panel.js` 中钩入了“DOM 拔插术”。在重绘前将播放中的音视频物理剥离至隐形金库 `#clab-mini-vault`，重绘后利用精准的时间戳比对 (`&t=`) 插回或强刷，实现 0 卡顿的无缝更新。
- **纯粹的数学排版引擎**：摒弃受制于浏览器绘制延迟的 `offsetWidth`，采用直接读取 CSS 变量推导的绝对数学布局，杜绝一切创建时的边缘错位与过渡闪动。
- **全局动态主题引擎**：引入纯 CSS 变量驱动的 UI 渲染方案，配合 `clab_settings.js` 实现无感、实时的个性化视觉定制。
- **面板事件分层**：`ui_panel.js` 负责组合与渲染；面板局部交互下沉到 `event_panel.js`；选择态逻辑沉淀到 `ui_selection.js`。

## 3. 核心机制设计图谱

### 🚀 3.1 无界阵列并发引擎 (Matrix Insertion)

在 `comp_toolbar.js` 中，“新建模块”按钮支持了倒序算法。如果用户通过 Shift 选取了分属于不同卡片的数十个模块，系统会将选中的索引收集并执行 `sort((a,b) => b-a)` 从下往上插入。保证了无论数组如何变动，所有新建的模块都会毫厘不差地出现在所选元素的正下方。

### 🎨 3.2 格式刷智能继承系统 (Smart Inheritance)

格式刷的设计初衷被重构为最符合人类直觉的逻辑：

- **提取时 (`action_module_config.js`)**：剥离旧历史记录，但**严格保留**输入模块的具体数值 (`value`)。
- **应用时 (`event_panel.js`)**：若目标卡片原本就有模块 A，则仅覆盖其排版与绑定规则，**绝不覆写**目标自带的数值；若目标坑位是刷出来的新模块，则 100% 顺延应用格式刷自带的初始数值。

### 🛡️ 3.3 全局媒体 404 防空洞 (Global Error Hijacker)

由于文件极易在后端 temp 目录被系统清理，我们在 `event_global.js` 的 `window` 捕获阶段装配了最高权重的 `error` 事件拦截器。
当且仅当发生真实的路径文件请求失败时，它会瞬间隐藏原媒体节点，并原地渲染出统一深黑底色（`#1e1e1e`）和红色警告图标的“媒体失效”界面。从而消灭了原生的浏览器裂图图标和干瘪的黑屏。

### ⚡ 3.4 纯数学布局渲染 (Math-based Layout)

为彻底解决新建/克隆卡片时由浏览器 DOM 渲染延迟导致的排版错位与闪跳问题，`comp_taskcard.js` 移除了所有 `setTimeout` 及 `offsetWidth` 判断。转而直接抓取 CSS 系统变量 `--clab-card-width`，配合纯公式 `count * width + gap` 提前计算好容器物理跨度。实现新建即就位，0 帧延迟的极速响应。

### 🧭 3.5 工作区/通道双层标签体系 (Workspace + Channel)

自 `schemaVersion = 3` 起，CLab 完成了“工作区 + 通道”双层隔离：

- **工作区层 (`comp_workspace.js`)**：管理整套卡片集合（cards）与运行上下文。
- **通道层 (`comp_channel.js`)**：在同一工作区下管理多套绑定（bindings）与工具栏状态快照（toolbarState）。
- **状态核心 (`ui_state.js`)**：
    1. `applyWorkspaceToState()` 切工作区时同步激活通道并回放绑定。
    2. `applyChannelToState()` 切通道时只切绑定视图，不重建业务数据。
    3. `syncStateToActiveWorkspace()` 在保存前把当前 area 绑定反向写回当前通道。
- **渲染挂载 (`ui_panel.js`)**：顶部工具栏下挂载通道栏，底部挂载工作区栏，形成两层标签导航。

## 4. 核心文件与函数字典 (File & Function Dictionary)

本章节依次详述每个文件的主要职责及其对外暴露的核心函数。

### 4.1 根目录文件 (Root Core)

### `web/main.js`

插件的最底层入口文件。

- **`app.registerExtension({...})`**: 向 ComfyUI 核心注册 `ComfyUI.CLab` 扩展。
- **`setup()`**: 在工作流环境就绪后，调用 `setupUI()` 初始化面板，并启动 API 劫持。
- **`loadedGraphNode()` / `nodeCreated()`**: 监听工作流加载及粘贴节点生命周期，触发数据智能恢复。

### `web/clab_settings.js` (新增)

全局设置的注册中心与动态主题引擎。

- 负责向 ComfyUI 原生设置面板挂载从“常规”到“外观主题”的全部配置项。
- 维护所有 `window._clab` 开头的全局设置状态。
- **`window._clabApplyTheme()`**: 计算色彩的 Alpha 通道并下发给 `:root` 的 `-clab-theme-` 系列 CSS 变量，实现所见即所得。

### `web/state_manager.js`

负责处理前端内存 (JS 对象) 与后端工作流 (ComfyUI 节点配置) 之间的数据双向绑定。

- **`StateManager.syncToNode(graph)`**: 先执行 `syncStateToActiveWorkspace()`，再通过 `buildPersistedState()` 序列化 `workspaces/channels` 结构并写入 `CLab_SystemConfig`。
- **`StateManager.loadFromNode(graph)`**: 页面刷新时，从节点参数中提取 JSON，反序列化后写入前端 `state` 并触发重绘。
- **`StateManager.syncLocalHistoryToOutputAreas()`**: 智能补全引擎，扫描硬盘历史记录填补丢失的模块历史。
- `state` 持久化主体已升级为 `workspaces[] + channels[]`，`cards` 仅作为当前活动工作区的运行态镜像。

### `web/api_injector.js`

网络拦截器与动态剪枝中枢。负责修改发送给后端的 Prompt 数据包。

- **`window.CLab.executeTasks(tasks)`**: 将待运行的任务数组加入 `_clabExecQueue` 并通过 `app.queuePrompt()` 依次触发。
- **`app.graphToPrompt` (被劫持重写)**:
    1. 拦截发包过程，将 `state` 里的输入参数 (Input Value) 深拷贝注入到对应的节点里。
    2. 启动“依赖树追踪算法”，剔除与目标“输出模块”无关的节点（动态剪枝）。
- **`api.addEventListener("executed")`**: 监听生成完成的钩子，触发 `/clab/copy_temp_asset` 接口将图片/视频转移至归档目录，并替换旧临时路径。

### `web/ui_panel.js`

主面板容器与组合入口，承载组件挂载、渲染编排与媒体保险库调度。

- **`window.CLab.stashMedia()` / `restoreMedia()`**: 重绘前拔出媒体元素藏入 `#clab-mini-vault`，重绘后插回，实现零闪烁防断连。
- **`setupUI()` / `createPanelDOM()`**: 生成悬浮面板的基础 HTML 与 CSS 样式。
- **`performPanelRender()`**: 渲染顺序为 `toolbar -> channel bar -> cards -> workspace bar`，并在前后保存滚动与媒体状态。
- 面板具体交互（选择、格式刷落点、滚轮、拖拽窗口、卡片宽度滑杆）已迁移到 `event_panel.js` 与 `ui_selection.js`。

### 4.2 视图组件层 (`web/components/`)

### `comp_taskcard.js`

任务卡片的生成器。

- **`updateCardsLayout()`**: 核心排版。放弃 DOM 测量，采用读取 CSS 变量的纯数学推导设置容器尺寸。
- **`generateSingleCardHTML(card, index)`**: 生成单张卡片的基础 HTML 骨架（带进度条和主题注入）。
- **`attachCardEvents()`**: 挂载改名、删除，并接管卡片级别的 Drag and Drop (拖拽排序) 中线计算逻辑。

### `comp_modulearea.js`

模块（Input/Output）的大管家，包含局部更新引擎。

- **`surgicallyUpdateArea(areaId)`**: 终极微创手术刀。锁定单个模块，单独替换其 `innerHTML`。内置独立保险库逻辑，精准保护不该重载的视频。
- **`updateAllDefaultTitles()`**: 魔法纠偏引擎。后台静默重算所有 `#1`, `##2` 的序号，防止删除或拖拽后序号断层。
- **`attachAreaEvents()`**: 接管模块拖拽逻辑（包含平行空间同步位移与跨卡片物理汇聚两大模式）。

### `comp_toolbar.js`

顶部静态工具栏的构建与交互。

- **`setupStaticToolbarEvents()`**: 管理静态工具栏事件（新建任务/模块、配置下拉、导入导出、运行入口接线）。
- 动态工具栏渲染与事件已委托给 `action_module_config.js`（本文件仅做转发导出）。

### `comp_workspace.js`

工作区标签栏（底部）的专属组件。

- **`renderWorkspaceBar(container)`**: 渲染工作区 tabs，带稳定 DOM 更新策略（减少重建，提升双击重命名稳定性）。
- **`attachWorkspaceEvents(container)`**: 挂载切换、右键菜单、新建/复制/删除、Shift/Ctrl 多选、拖拽排序、双击重命名。
- **`switchWorkspace()`**: 切换前先 `syncStateToActiveWorkspace()`，切换后 `applyWorkspaceToState()`，确保跨工作区数据不串。

### `comp_channel.js`

通道标签栏（顶部工具栏下方）的专属组件。

- **`renderChannelBar(container)`**: 渲染当前工作区下的通道 tabs。
- **`attachChannelEvents(container)`**: 挂载切换、右键菜单、新建/复制/删除、Shift/Ctrl 多选、拖拽排序、双击重命名。
- **`switchChannel()`**: 调用 `applyChannelToState()` 回放目标通道绑定，实现同工作区内多绑定方案切换。

### `comp_contextmenu.js`

全盘右键菜单管理。

- **`setupContextMenu()`**: 接管全局 `contextmenu` 事件，获取鼠标点击的 `areaId`。
- 提供局部操作入口：单模块清理 404 死链、单模块下载 ZIP、删除同名模块、批量前后推移。

### `ui_state.js` & `ui_selection.js`

- **`state` / `dragState`**: `state` 已包含 `workspaces/channels` 双层结构，`cards` 是当前活动工作区镜像；`dragState` 存储拖拽状态。
- **`saveAndRender()`**: 数据变动后触发 `clab_render_ui` 渲染事件。
- **`applyWorkspaceToState()` / `applyChannelToState()`**: 两级标签切换入口，负责绑定回放与当前通道激活。
- **`syncStateToActiveWorkspace()`**: 保存前把当前 `cards` 与绑定回写到活动工作区/活动通道。
- **`handleSelectionMouseDown()`**: 单选/Ctrl 多选/Shift 区间多选的主入口（卡片与模块）。
- **`handleDeselectAll()`**: 清空当前选择并处理 painter 退出。
- **`updateSelectionUI()`**: 局部更新卡片/模块高亮、刷新动态工具栏，并静默保存状态。

### `ui_utils.js`

提供跨文件通用的纯函数。

- **`buildCustomSelect()`**: 生成带搜索过滤功能的精致下拉框 HTML。
- **`getWidgetDef(nodeId, widgetName)`**: 解析图谱，获取目标节点特定参数的原始类型。
- **`showBindingToast()`**: 显示屏幕中上方的透明悬浮提示框。

### 4.3 交互控制层 (`web/components/actions/`)

### `action_module_config.js`

动态悬浮工具栏引擎，面板操作最密集的区域。

- **`renderDynamicToolbar()`**: 根据当前选中动态生成包含比例配置、节点拾取、模式切换的工具栏 HTML。
- **克隆逻辑 (`#tb-clone-btn`)**: 深度拷贝选中对象并利用 `DocumentFragment` 物理插入 DOM，100% 携带所有历史记录。
- **格式刷逻辑 (`#tb-format-painter`)**: 深度提取格式，剥除所有历史记录图库，但**保留**输入模块的值。

### `action_batch_sync.js`

强大的跨卡片批量操作处理器。

- **`execSyncParams()`**: 遍历全盘，将当前参数强制同步给所有具有同名标题 (`##X`) 的兄弟模块。
- **`execMoveBackward()` / `execMoveForward()`**: 获取目标元素后，利用原生 `appendChild` 机制在 DOM 树中进行无感物理拽移。

### `action_data_io.js`

文件与剪切板的进出口管线。

- **`processImportedJSON()`**: 将剪切板或本地 JSON 解析，根据模式分发为“新建”或“追加”。
- **`downloadMediaFiles()`**: 引入 `JSZip` 将收集到的媒体链接打包为压缩包并拉起浏览器下载。
- **`organizeOutputFiles()`**: 通过调用 `/clab/organize_files`，实现跨目录的文件移动与整理，自动读取并应用 `clab_settings.js` 中设定的 Archive 目录。

### `action_binding.js`

- **`enterBindingModeForSelected()`**: “拾取节点”的实现。隐藏面板，接管画布鼠标事件，支持智能穿透 `Primitive` 基元节点寻找真实参数。

### 4.4 事件总线层 (`web/components/events/`)

### `event_global.js`

维护系统的“生命体征”。

- **全局 Error 拦截器**: 嗅探到媒体加载 404，立刻使其隐身并原地铺设丢失兜底 UI。
- **多选护盾**: 拦截不合时宜的 `mousedown`，保护 `Shift/Ctrl` 多选阵列在拖拽前不被意外清空。
- **键盘监听**: 动态计算修饰键支持快捷键调出，`Escape` 退出模式，方向键无缝切换图库。

### `event_execution.js`

- **`setUIProgress()`**: 进度条控制器。维护各个卡片的专属进度条，并配合 `_clabHaltOnError` 全局变量智能阻截或跳过后续出错任务。

### `event_panel.js`

- **`setupPanelEvents(panelContainer)`**: 面板局部交互总入口。
- 负责：批量运行次数输入、面板拖拽、卡片宽度滑杆、卡片区滚轮滚动、格式刷落点处理、媒体错误回退钩子。
- 与 `ui_selection.js` 协作：由 `event_panel` 统一监听事件，由 `ui_selection` 执行选择态算法与 UI 更新。

### 4.5 模块渲染层 (`web/components/modules/`)

### `module_input.js`

- **`generateInputHTML()`**: 渲染文本区、布尔开关或拖拽上传区，并无缝支持 `media_video.js` 及其默认静音与自动播放设置体系。

### `module_output.js`

- **`generateOutputHTML()`**: 生成全屏媒体预览，或者 `display: grid` 的四宫格历史记录陈列。
- 支持读取 `window._clabThumbPerfMode` 动态切换“截取首帧”或“自动静音循环”的缩略图渲染策略。

### `module_media.js` & `media_types/*.js`

多媒体底层渲染组。

- **`media_video.js`**: 视频专员。提供包含防抖高精度鼠标滚轮快进算法、抽屉式音量滑杆的高阶播放器。
- **`media_audio.js`**: 音频专员。提供利用单例 `requestAnimationFrame` 驱动的高频顺滑进度条。

### 4.6 后端接口定义 (`api.py`)

Python 层原生操作模块。

- **`CLabSaveImage` (类)**: 继承并重写官方 SaveImage。修复恶心的下划线尾缀，实现了 `pix_01.png` 标准防覆盖递增算法。
- **`/clab/delete_file`**: 接收前端路径，安全校验目录穿越 (Directory Traversal) 后，执行 `os.remove`。
- **`/clab/organize_files`**: 根据请求清单，调用 `shutil.move` / `shutil.copy2` 进行物理梳理。
- **`/clab/copy_temp_asset`**: **截胡中枢**。被前端调用将临时文件转移到专属 `CLab/` 目录，支持接收前缀定制和 `delete_temp` 删除原缓存的指令。

### 4.7 文字抓取与 TXT 持久化链路（新增）

本次新增的“文字”输出能力，与图片、视频、音频、3D 文件一样，已经进入统一的输出模块体系；不同点在于，文字历史记录以本地 `.txt` 文件为最终事实来源，而不仅仅依赖前端内存缓存。

- **抓取入口 (`web/api_injector.js`)**
  - 在 `api.addEventListener("executed")` 中，除了继续处理 `images / gifs / videos / audio` 等资产，也会尝试抽取当前执行节点返回的文本。
  - `extractTextFromOutputData(outputData)` 会优先从后端返回值里读取 `text / texts / string / strings / markdown / value` 等字段。
  - `extractTextFromNodeWidgets(node)` 会回退读取节点实例上的 widget 值，因此兼容 `PreviewAny` 这类把文本写进前端预览控件的节点；对于 `preview_text`、`preview_markdown` 这类 widget，同样可以被识别。

- **落盘接口 (`api.py`)**
  - `POST /clab/save_text` 负责把抓取到的文本保存到 `ComfyUI/output/CLab/text/` 目录。
  - 命名规则与图片/视频归档一致，默认采用 `pix_编号.txt` 递增命名。
  - `archive_dir` 与 `file_prefix` 仍沿用现有 CLab 配置，因此如果用户在 `⚓ CLab 系统配置` 中修改归档目录或前缀，文本也会跟随同一规则落盘。
  - `GET /clab/get_local_history` 当前保留为兼容接口。

- **历史记录结构 (`web/components/modules/media_types/media_utils.js`)**
  - `pushPreviewHistoryEntry(area, url, { kind: "text", text })` 会把文本记录写入 `area.history`，并同步维护 `area.textHistory` 与 `area.textHistoryStatus`。
  - `syncTextContentWithSelection(area)` 负责把当前 `historyIndex` 指向的文本内容同步到主预览区。
  - `textHistoryStatus` 目前使用 `idle / loading / ready / missing / error` 五种状态，用于区分未加载、加载中、已就绪、文件丢失和读取失败。

- **重启恢复与重新同步 (`web/state_manager.js`, `web/components/comp_toolbar.js`, `web/components/comp_contextmenu.js`)**
  - `StateManager.loadFromNode()` 完成面板状态恢复后，会调用 `reloadTextHistoriesFromFiles()`，批量执行 `loadAllTextHistory(area, { force: true })`，从本地 txt 文件重新读取内容。
  - 这意味着 ComfyUI 重启后，CLab 会根据历史记录中的文件 URL 重新加载文本，而不是继续使用上次运行时残留的内存内容。
  - 顶部配置菜单与右键菜单中的“重新同步记录”都会强制重新读取本地 txt；如果用户在外部编辑器中修改了文件，刷新后这里显示的就是最新文本。

- **缺失态与清理失效记录**
  - `loadTextHistoryEntry()` 在读取 txt 时如果遇到 `404`，会把对应条目标记为 `missing`。
  - 预览组件会在 `missing` 状态下显示“媒体丢失”。
  - “清理失效记录”会扫描这些失效条目，并把它们从模块历史中移除，不会误删仍然存在的有效 txt 记录。

- **预览渲染 (`web/components/modules/media_types/media_text.js`, `web/components/modules/module_media.js`, `web/components/modules/module_output.js`)**
  - `module_media.js` 已把 `renderText / attachTextEvents` 纳入统一媒体路由；因此输出模块切换到文本结果时，仍然走同一套媒体分发机制。
  - `media_text.js` 提供文字专用预览壳层，当前包含：
    - 文字内容使用无衬线字体显示。
    - 预览背景为 `rgb(25,25,25)`，文本框背景为 `rgb(32,32,32)`。
    - 选项栏位于文本预览区下方，并保留顶部空白给左上角历史数量标签。
    - “语法高亮”：对 JSON / 代码风格文本做轻量关键字、数字、字符串、注释高亮。
    - “预览Markdown”：以简化 Markdown 规则渲染标题、列表、引用、代码块、链接与行内样式。
    - “复制”按钮：复制当前显示的完整文本内容。
    - 自适应高度：当文本总高度不大于模块宽度时，预览区按内容高度收缩；超过后固定为“模块宽度对应的高度”，并启用细滚动条。
  - `module_output.js` 在管理模式下会为文本历史生成摘要缩略卡，支持与图片/视频历史一起切换、删除、清理和重新同步。

- **左右方向键切换历史**
  - `web/components/events/event_global.js` 在左右方向键切换输出历史时，会同步更新 `historyIndex`、`resultUrl`，并立即调用 `syncTextContentWithSelection()` 与 `loadSelectedTextContent()`。
  - 因此文本历史的切换行为现在与图片/视频历史一致，但真正显示内容始终以对应 txt 文件重新读取后的结果为准。
