### [v0.2.5] - 2026-03-08

- **修复**: 工具栏“新建任务”按钮在没有任何选中时新建卡片，会导致卡片后面的“新建任务卡片”不会自动向后移动（应该是要始终跟着所有卡片的后面，但是现在还在原本的位置，需要关闭面板——打开面板才会恢复，是不是局部更新没有应用到它）
    - **解决方案**: 去除后面的大按钮，并清理工具栏的幽灵引用 (`comp_toolbar.js`)、清理克隆模块的幽灵引用 (`action_module_config.js`)、清理 JSON 导入的幽灵引用 (`action_data_io.js`)
- **优化**: 克隆按钮：不能克隆历史记录。格式刷在复制的时候可以把生成记录克隆过去。反过来，让克隆功能可以克隆历史生成记录，格式刷在复制卡片、模块的时候，仅能复制参数和值。
- **优化**: 在面板滚动条还没出现的时候，创建的卡片会居中对齐。目前居中对齐的操作好像有些快速操作的闪动依稀能看到，比如：新建第一张卡片的时候，似乎是现在左边创建卡片，然后有快速的居中对齐；其它的也类似，新建第二种的时候，似乎在最右边创建，然后快速移动到最后的卡片旁边，与其它卡片居中对齐。当滚动条出现后再新建卡片、克隆卡片时，又没有这种闪动的感觉了。这是怎么回事？有办法优化不？
- **修复**: “同步参数”按钮：模块绑定带有组件的参数时，把参数同步到其它卡片相同名称的参数后，同步过去的模块不能显示组件（视频组件、音频组件等）
- **修复**: 格式刷按钮：与“同步参数”一样，当模块绑定带有组件的参数时，使用格式刷把其他模块都同样绑定这个参数，这时格式刷刷过的模块同样不能显示组件（视频组件、音频组件等）。有些时候可以显示，有些时候又不能显示了。需要检查是不是重启ComfyUI的缘故。
- **问题**: 不同工作流切换时，其它工作流内这个插件的记录会被浏览器内存清理吗？当打开多个工作流同时使用时，会不会很卡？

### [v0.2.4] - 2026-03-07

- **修复**: 继续修复因为把“全量渲染”更改为“局部更新”而产生的bug
- **优化**: 拖拽模块
    - 拖拽模块同步：选中不同卡片的模块在内部向上拖动时，第一次会把所有卡片都会同步向上移动，但是接下来第二次拖动时，就仅会拖动当前卡片内的模块了。
    - 跨卡片汇聚：选中不同卡片的模块到其它卡片时，不能把所有卡片汇聚在目标卡片了。
    - 选择单个模块，也不能把选中的模块拖拽到其它卡片内了。（也就是说，不能再不同卡片内部相互拖动模块了），甚至当把模块拖拽到其它卡片（实际还在当前卡片内），在当前卡片内拖拽都不能更改位置了。
- **修复**: 当选中多个模块后，再次点击“选中的其中一个模块”的问题：
    - 当选中多个模块后，再次点击选中模块空白处，会单选这个选中的模块；
    - 但是在选中多个后，再点击选中模块的“参数输入框”、“标题输入框”，就不能单选这个模块了。
- **修复**: 新建模块后，模块的“隐形锚点”没有跟随新建模块
- **优化**: 切换模块后，模块编号不能自动局部刷新
- **修复**: “同步参数”按钮：模块绑定带有组件的参数时，把参数同步到其它卡片相同名称的参数后，同步过去的模块不能显示组件（视频组件、音频组件等）
- **修复**: 格式刷按钮：与“同步参数”一样，当模块绑定带有组件的参数时，使用格式刷把其他模块都同样绑定这个参数，这时格式刷刷过的模块同样不能显示组件（视频组件、音频组件等）
- **优化**: “重置”按钮：在重置的时候，仅重置模块工具栏的参数，不要重置标题和输入的参数值。
- **优化**: 克隆按钮：在克隆模块后，目前选中还是的原本的模块。修改为：选中新克隆的模块（就像克隆卡片那样，选择新克隆的卡片）
- **修复**: 克隆卡片：克隆按钮，在克隆卡片的时候还是会闪动（在克隆模块的时候就不会了）
- **修复**: 面板滚动条：删除卡片，导致卡片数量减少到滚动条消失后，剩余的卡片不能居中对齐。
- **修复**: 面板滚动条：当卡片数量超出面板范围时，会出现面板滚动条。现在面板有两个问题：
    - 在没有出现面板滚动条的时候，所有卡片在面板上水平居中对齐，但是目前居中对齐把卡片后面的“新建卡片”按钮也放进去了。（不用放进去，居中对齐仅把卡片居中对齐）
    - 当删除卡片，导致卡片数量减少到滚动条消失后，剩余的卡片是左对齐，不是居中对齐了（是不是因为滚动条消失后没有局部刷新？）
- **优化**: 右键菜单
    - 执行“选择相同模块”还是会闪动，也修改为局部更新。
    - 批量向后移动：在批量向后移动时，如果最后面没有卡片时，会创建一个卡片移动。也就是说当在创建卡片时会全局刷新闪动（在没有创建时就不会闪动），能不能修改为：局部更新（因为现在新建卡片都不会全局刷新了）
- **优化**: 导入：所有的导入选项（创建任务、追加模块、追加模块到选中），在导入时还是会刷新全部，这个你建议局部刷新不？
- **优化**: 模块默认标题自动编号的问题：区分输入模块和输出模块（比如：在给输入模块默认编号的时候，如果中间有输出模块，就自动跳过这个输出模块，给到下一个输入模块；输出模块同理）
- **优化**: 选择相同模块：是通过标题名称，在所有卡片上选择名称相同的模块。当没有自定义名称时，就使用默认名称##1、##2…等来查找相同模块。
- **修复**: 新工作流中，新建模块后模块不能拖动的bug
- **修复**: 模块的平行空间同步拖拽，拖拽的模块需要多走一个的问题
- **优化**: 在“输出模块”的工具栏——关联节点选项菜单自动关闭（点击空白处关闭）
- **修复**: 输出模块当“匹配媒体比例”取消勾选后，视频组件、音频组件的操作会失效。并且重新勾选后，也需要来回切换一次生成记录才会生效。

### [v0.2.3] - 2026-03-06

- **重构**: 为了修复在操作时，“输出模块”上视频闪烁的问题，把操作时“全量渲染”更改为“局部更新”逻辑。
- **代码改进**:
    - `comp_modulearea.js`: 模块增删改；暴露定点更新引擎（其他文件调用，打破文件循环依赖）；标题默认编号自动纠偏；在 surgicallyUpdateArea 内部实现了点穴级的“拔插”逻辑，并让序号不仅对编辑模块生效，也会无缝覆盖到预览模块。
    - `comp_taskcard.js`: 卡片增删改；标题默认编号自动纠偏。
    - `comp_toolbar.js`: 工具栏全局创建的局部渲染。
    - `action_module_config.js`: 工具栏参数调节；修复“管理记录”的高亮缺失与工具栏闪烁；修复工具栏下拉菜单的自我刷新与状态联动。
    - `module_output.js`: 生成记录管理。
    - `comp_contextmenu.js`: 移除；清除记录；清理失效记录；重新同步记录。
    - `event_global.js`: 左右方向键切换刷新。
    - `ui_panel.js`: 让格式刷操作实现“点穴级”覆盖；顶部注入了 `stashMedia` 引擎，彻底保护格式刷等全量重绘不闪烁。
    - `module_input.js`: 布尔值、选项、加载图片、加载视频、加载音乐、加载文件等接入“局部更新”引擎。
    - `ui_utils.js`: 修改了下拉框的值，我们就只重新渲染这一个模块。
    - `api_injector.js`: 修复运行后的全局闪烁。
    - `action_batch_sync.js`: 同步参数；删除相同；批量位移。
