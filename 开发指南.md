ComfyUI ShellLink 开发者指南

本文档面向对 ShellLink 插件进行二次开发或维护的工程师，详述了组件化重构后的文件职责、系统架构以及核心函数的调用说明。

1. 架构概述

ShellLink 采用了完全解耦的组件化前端架构（MVC / 乐高模式）。数据层 (Model)、视图层 (View) 和控制交互层 (Controller / Actions) 被严格分离：

数据流向：视图组件修改 ui_state.js 中的中心化数据 -> 调用 saveAndRender() -> 触发 DOM 局部重绘 -> 将 JSON 序列化并存入 ComfyUI 画布上的 ShellLinkSystemConfig 节点。

视图与交互分离：components/ 目录下的组件（如卡片、模块、菜单）尽量只负责“长什么样”（DOM 渲染与原生拖拽），而实际的业务计算（如导出导入、批量同步、编译运行队列）全部分发给 components/actions/ 目录下的专用控制器文件，实现“一份逻辑，随处挂载”。

执行拦截：api_injector.js 劫持 app.queuePrompt。在向 Python 后端发送请求前，从 state 中读取参数，深拷贝并修改 Node 的 inputs，最后投递执行。

3. 核心文件详解与函数清单

🔧 1. 核心与底层拦截层 (web/)

web/main.js

插件的注册入口，统筹各模块的初始化。

setup(): 扩展加载入口，调用 UI 初始化与 API 劫持。包含清空图谱时重置面板的钩子。

loadedGraphNode(node) / nodeCreated(node): 监听配置节点事件，实现跨工作流复制与数据恢复。

web/api_injector.js

整个插件最核心的“偷天换日”逻辑，负责将 UI 参数注入底层执行流。

setupAPIInjector(app): 挂载 API 劫持与全局函数。

window.ShellLink.executeTasks(tasks): 核心调度方法。根据任务队列，对 Prompt 进行动态剪枝、参数替换，并将 PreviewImage 等原生节点替换为插件专属的 ShellLinkSaveImage 节点以规范文件命名。

web/state_manager.js

管理前端内存与 ComfyUI Canvas 物理节点的数据双向同步。

startWatchdog(graph): 轮询看门狗，侦测外部引起的节点数据变化（如撤销、粘贴）。

syncToNode(graph) / loadFromNode(graph): 将 ui_state 的数据与画布上的隐藏 STRING 端口进行互传。

web/ui_panel.js

主面板的最高层容器，负责组装所有组件和挂载原生通信事件。

setupUI() / createPanelDOM(): 构建主面板 DOM 外壳，注入通用 CSS 样式，并接管全局点击与滚动逻辑。

setUIProgress(cardId, percentage): 核心 UI 进度条引擎，驱动卡片标题下方的进度条动画与报错飘红。注意：进度条现已从各个模块移至任务卡片级别。

setupGlobalEventListeners(): 深度监听 WebSocket (如 execution_start, progress, executed) 以驱动状态流转，处理返回的图片 URL。

enterBindingModeForSelected(): 开启沉浸式拾取模式，拦截 pointerup 以实现一键绑定画布节点。

🎨 2. UI 视图组件层 (web/components/)

ui_state.js & ui_utils.js

state / dragState / appState: 纯前端的状态仓库，存储选中项、拖拽标记与格式刷源。

saveAndRender(): 被所有 UI 交互调用，持久化状态并派发 sl_render_ui 局部重绘事件。

buildCustomSelect(): 统一的定制化下拉菜单构建器。

getWidgetDef() / getCustomNodeMenuHTML(): 深入遍历 ComfyUI 内存图谱，解析目标节点及其支持的 Widget 参数树。

comp_toolbar.js

极度瘦身的静态 UI 外壳。仅负责通过 setupStaticToolbarEvents 组装全局的“新建”、“运行”、“导入导出”等按钮，并将实际事件委托给 actions/ 下的控制器。

comp_taskcard.js

renderCardsList(): 渲染卡片列表结构。

包含基于 HTML5 DnD 的中线判定逻辑 (Midline Detection)，精准判断卡片拖拽的前插或后插落点。

comp_modulearea.js

generateAreaHTML(): 承担繁重的模块内部 UI 渲染（包括多种数据类型的输入框、文件上传热区、单图预览模式、以及多历史记录的网格管理模式）。

attachAreaEvents(): 处理模块的选中、原生文件拖拽上传，以及缩略图级别的多选、批量 DnD 拖动排序逻辑。

⚙️ 3. 动作与交互控制器层 (web/components/actions/)

action_module_config.js

renderDynamicToolbar(): 根据当前选中的模块类型（Input/Output）和数量，动态生成悬浮工具栏的 HTML。

attachDynamicToolbarEvents(): 绑定工具栏事件，包括节点关联、比例切换、一键同步、克隆以及极其强大的格式刷 (Format Painter) 功能。

action_batch_sync.js

跨模块操作的核心引擎。

execSyncParams(): 将当前模块参数一键广播给其它所有卡片同位置的模块。

execDeleteSameModules(): 批量清除同名标识模块。

execMoveBackward() / execMoveForward(): 跨卡片切片的批量向上/向下移动。

action_run_executor.js

buildTasksQueue(): 核心编译器。根据用户“选中模块 -> 选中卡片 -> 未选中任何”的严格优先级，将零散的状态编译为结构化的单卡片或多卡片执行队列 (Tasks Queue)。

action_data_io.js

processImportedJSON() / handleClipboardImport(): 处理外部 JSON 的反序列化，支持智能追加和新建卡片。

downloadMediaFiles(): 批量打包并调用 JSZip 将目标模块的当前或所有历史文件压缩下载。

organizeOutputFiles(): 获取收集指令，向后端 API 发送跨目录的移动或复制请求，并同步更新前端所有 history 中的指向地址。

🖥️ 4. 后端 Python 接口 (api.py)

为了突破 ComfyUI 的原生限制，ShellLink 注入了自己专属的系统级 API：

ShellLinkSaveImage (Node): 动态注册的“无下划线”图片保存节点。拦截替换掉原版的 SaveImage，强制去除 _ 后缀，使得输出文件呈现完美的 Pix_00001.png 格式，并带有强力 while 防覆盖自增机制。

/shell_link/delete_file (API): 配合前端的“销毁本地文件”操作，彻底删除指定目录下的产出垃圾。

/shell_link/organize_files (API): 本地资源归档引擎。将散落在各处的产出文件，物理移动（或复制）到以配置节点名称命名的统一子文件夹中，以实现项目级的资产隔离。
